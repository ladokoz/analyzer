# Ahub Video Analyzer

Ahub Video Analyzer is an internal, web-based AI tool that leverages the Gemini API to analyze videos from YouTube and Vimeo. It can extract rich metadata such as directors, producers, film genres, technical animation styles, and accessibility attributes by semantically analyzing video content based on custom robust prompts.

## Features

- **Batch Processing**: Process multiple YouTube and Vimeo URLs simultaneously in a background thread.
- **Vimeo Support**: Natively downloads and processes Vimeo videos (even private ones with valid credentials).
- **Gemini API Integration**: Built-in support for Gemini Flash and Pro models (including Gemini 3.1 Preview).
- **Automated CSV Exports**: Automatically aggregates extracted video data into timestamped, Excel-compatible CSV batches (UTF-8 with BOM).
- **Persistent Queue**: A robust backend queuing system prevents active tasks from halting if the browser is closed.
- **Robust Environment Security**: Sensitive API keys and credentials are natively managed via a root `.env` system, avoiding exposure through the web UI.
- **Web-Based Updater**: Push-button updates from the Settings UI to pull the latest version from GitHub.

## Installation & Setup

### Requirements

- **Python 3.10+**
- **FFmpeg**: Strongly recommended to be installed locally and injected into your system path for handling complex video downloads via `yt-dlp`. Linux users can run `sudo apt install ffmpeg`.
- **Git**: Required for the automated update mechanism.

### 1. Repository Access & Git Setup
For the **Update** button to function, the project directory **must** be a valid Git repository.

**Option A: New Installation (Recommended)**
```bash
git clone https://github.com/ladokoz/analyzer.git
cd analyzer
```

**Option B: Existing Manual Upload**
If you uploaded files manually, initialize git and link it to the repository:
```bash
git init
git remote add origin https://github.com/ladokoz/analyzer.git
git fetch
git reset --hard origin/main
```

### 2. Environment Configuration (.env)

The application secures all critical information—including login variables—using a `.env` file. 

Create a file named exactly `.env` in the root folder of the project (next to `run.bat` and `run.sh`). Open it in a text editor and add the following lines:

```ini
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_secure_password
GEMINI_API_KEY=AIzaSy_your_gemini_api_key_here
VIMEO_USERNAME=your_vimeo_email@example.com
VIMEO_PASSWORD=your_vimeo_password
```

> **Note**: Vimeo credentials and Gemini API keys are optional initially, but the system will be non-functional until a valid Gemini API key is linked.

### 3. Launching the Application

The analyzer includes an automated launch script that handles virtual environment creation, pip installations, and deploying the FastAPI server.

**On Windows:**
Double-click `run.bat` or run it from the command prompt:
```cmd
.\run.bat
```

**On Linux (Debian / Proxmox server):**
Give execution permissions and run the script:
```bash
chmod +x run.sh
./run.sh
```

### 4. Accessing the UI

1. Open your web browser.
2. If running locally on Windows: Navigate to `http://localhost:8000`
3. If running on a Linux server/container: Navigate to `http://<your_server_ip>:8000` (or via your configured Cloudflare Proxy domain).
4. Log in using the `ADMIN_USERNAME` and `ADMIN_PASSWORD` you configured in your `.env` file.

## Modifying Settings & Updates

Once logged in, click the **⚙️ Settings** button near the top right. Here you can configure:
- **App Version & Updates**: View the current version (e.g., `v1.2.0`) and click **Check Updates** to pull the latest changes from GitHub automatically.
- **LLM Model Preferences**: Select between different iterations of the Gemini architecture (e.g., Gemini 2.0 Flash or Gemini 3.1 Pro previews).
- **Token Price Tracking**: Control your current `Input` and `Output` cost metrics to automatically log analysis costs to the resulting CSV datasets.
- **Prompt Library**: Create, edit, and toggle between specific instruction prompts designed to extract unique metadata from video visual/audio analysis.

*(Note: API Keys and Vimeo credentials are managed strictly from the `.env` file for enhanced security against front-end intrusions).*

## File Architecture

- `backend/main.py`: The core FastAPI routing and server backend logic.
- `backend/worker.py`: Background threaded queue pipeline to prevent browser disconnects or refreshes from interrupting active API calls.
- `backend/gemini_utils.py`: API payload mappings communicating directly to Google's GenAI module.
- `backend/config.py`: Local persistent state configurations (managing prompts and settings objects).
- `frontend/`: Pure HTML, JS, CSS. Fully custom internal web application interface.
- `data/`: Autogenerated persistent data directory to capture runtime application logs, video file buffers, and export output CSVs.
